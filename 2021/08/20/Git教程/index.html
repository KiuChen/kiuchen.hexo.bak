Git教程

发表于 2021-08-20 更新于 2021-12-28 分类于 git

  1. 初始化仓库：git init 仓库名

  2. 添加文件到仓库：git add 文件名

    错误fatal: pathspec 'readme.txt' did not match any files：说明该文件不存在

  3. 提交文件到仓库：git commit -m “wrote a readme file”(-m xxx是说明，方便别人阅读)

    执行后会返回：1 file changed 一个文件被更改, 2 insertions(+) 插入两行内容

    add和commit：commit可以一次提交多个文件，add多次多个。

  4. 查看仓库当前状态：git status

    在add指令后输入git status查询状态，会告知要提交的修改包括哪些文件。

    在commit指令后输入git status查询状态，会告知 nothing to commit, working tree clean

  5. 查看文件修改了哪些内容：git diff xxx(带格式后缀的文件名)

  6. 查看历史记录：git log(显示从最近到最远的提交日志)--pretty=oneline

    开头部分的英文+数字是commit id(版本号)，标记了HEAD->master的是当前版本。

  7. 回退到上个版本：git reset –-hard HEAD^

    该命令返回HEAD is now at 8f4d43a(版本号) add distributed(版本说明)

    回退到指定版本：git reset –hard 1094adb(版本号)

  8. 查看命令记录：git reflog记录每一次命令

  9. 工作区和暂存区：

    电脑能看到的目录，就是一个工作区，隐藏目录.log是git版本库，里边存有一个暂存区， 还有git为我们自动创建的第一个分支master，以及指向master的一个指针HEAD。

    git add把文件添加到暂存区，git commit把暂存区的所有内容提交到当前分支后，暂存区被清空。

  10. git restore 文件名：撤销修工作区文件修改的内容

    git restore –stage 文件名：撤销git add命令的动作(将文件修改移除暂存区)

  11. 删除文件：git rm 文件名(从版本库删除该文件) git commit -m “描述”。

    注：从未添加到版本库就被删除的文件，是无法恢复的。

  12. 使用ssh远程连接git仓库：

    git remote add origin(远程库名称) git@github.com:KiuChen(github/learngit(要远程连接的仓库).git

  13. 从本地向远程库添加：

    git branch -M main

    git remote add origin git@github.com:KiuChen/learngit.git

    git push -u origin main

  14. 从远程克隆到本地：

    git clone git@github.com:KiuChen/gitskills.git

  15. HEAD指向master/分支来的到当前版本。

    创建分支dev:git checkout -b(表示创建并切换) dev

    等效于：(1)git branch dev 创建(2)git switch dev 切换

  16. 查看当前分支：git branch，会列出所有分支，带*的是当前分支。

    合并指定分支到当前分支：git merge dev(指定分支)

    禁用fast forward：git merge --no-ff -m "merge with no-ff" dev

    删除分支：git branch -d dev

    查看分支合并情况：git log –graph

  17. 保存当前工作现场：git stash

    删除工作现场：git stash pop

  18. 复制指定版本号的提交合并：git cherry-pick <commit>

  19. 多人协作的工作模式通常是这样：

    (1) 首先，可以试图用git push origin <branch-name>推送自己的修改；

    (2) 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；

    (3) 如果合并有冲突，则解决冲突，并在本地提交；

    (4) 没有冲突或者解决掉冲突后，再用git push origin <branch-name>推送就能成功！

    如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream-to <branch-name> origin/<branch-name>。

  20. 打一个新标签：git tag <name>

    查看所有标签：git tag 标签名 版本号id

    带说明的标签：git tag -a 标签名 -m “说明文字” 版本号

    注：标签和commit挂钩

    删除标签：git tag -d 标签名

    推送某个标签到远程端：git push origin <tagname>

    推送全部尚未推送的标签到远程端：git push origin –tags

    删除远程的某个标签：首先git tag -d 标签名 然后git push origin :refs/tags/标签名
